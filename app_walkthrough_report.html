Scenario 2 — Student Tutor Flow (Documented)

2.1 Subject → Chapter → Topic Selection

📋 DOCUMENTED

The core student journey flows through three hierarchical selection screens. After login, the student sees subject cards (e.g., Mathematics, Science, English). Clicking a subject shows its chapters. Clicking a chapter shows its topics. This three-level hierarchy allows students to precisely pick what they want to study.

Expected User Journey (per e2e scenarios.json)

1Navigate to / (home — subject list)
2Wait for [data-testid='subject-item'] to appear
3Click a subject → chapter list loads
4Click a chapter → topic list loads
5Click a topic → mode selection screen appears
What to expect: Each level shows a scrollable list of cards. The app uses [data-testid] attributes for e2e testability. The backend API serves curriculum data (subjects/chapters/topics) from the Aurora PostgreSQL database.

2.2 Learning Mode Selection

📋 DOCUMENTED

After selecting a topic, the student is presented with three learning mode cards. This is a key design decision point in the student experience. A "Resume" option also appears if the student previously paused a Teach Me session on this exact subtopic.

🎓 Teach Me Mode

The tutor explains the topic step-by-step from scratch. Tracks progress and coverage percentages. Can be paused and resumed later. The tutor proactively asks questions to check understanding before moving on.

💬 Clarify Doubts Mode

The student asks their own questions about the topic. The tutor answers and tracks which concepts were discussed. Good for targeted review of specific areas of confusion.

📝 Exam Mode

The tutor quizzes the student with questions from the topic. Tracks correct answers. Can be ended early to see partial results. After finishing, shows a detailed exam review with question-by-question breakdown, correct answers, and explanations.

The mode selection screen uses [data-testid='mode-teach-me'], [data-testid='mode-clarify-doubts'], and [data-testid='mode-exam'] attributes, confirming all three modes are implemented.

2.3 Live Tutor Chat Session (Teach Me)

📋 DOCUMENTED

Once a mode is selected, the student enters the chat interface. The tutor opens with a welcoming message and begins teaching. The interface is a mobile-first chat view with the AI tutor's avatar, message bubbles, and an input area at the bottom.

  • Chat Input: Text field ([data-testid='chat-input']) for typing responses
  • Send Button: [data-testid='send-button'] to submit
  • Mic Button: [data-testid='mic-button'] for voice input (speech-to-text)
  • Pause Button: [data-testid='pause-button'] to pause session and come back later
  • Teacher Avatar: Animated GIF showing the tutor character
  • TTS: Text-to-speech in English, Hindi, or Hinglish via Google Cloud TTS API
  • Progress Tracking: Coverage percentage displayed for Teach Me sessions
Backend architecture: The tutor pipeline uses a master tutor LLM agent (currently configured with GLM-5 model via ccapi.ai). The pipeline includes safety filtering, context management, and adaptive response generation. The system uses Server-Sent Events (SSE) for real-time streaming of tutor responses.

3.2 Books Dashboard — Curriculum Upload

📋 DOCUMENTED

The Books admin panel allows admins to upload textbook content and let the AI extract the curriculum structure. This is the pipeline for adding new content to the system.

Admin Book Ingestion Workflow

1Navigate to /admin/books — view all books dashboard
2/admin/books/new — Create new book (upload PDF pages)
3AI extracts table of contents from uploaded pages
4Process chapters — AI extracts topics, guidelines, study plans
5Sync to curriculum — push processed topics into live curriculum
Uses [data-testid='create-book-form'] — the form allows specifying the book name, grade level, subject, and syllabus (e.g., CBSE Grade 3 Mathematics).

3.3 Evaluation Dashboard — Tutor Quality Testing

📋 DOCUMENTED

The Evaluation dashboard (/admin/evaluation) enables administrators to test tutor quality using simulated student personas. The system uses an "eval_evaluator" LLM to score tutor responses on 7 quality dimensions.

  • Run evaluation with a simulated student persona
  • View scoring across 7 quality dimensions
  • Compare performance across different LLM configurations
  • Review conversation transcripts from eval runs
Evaluation dimensions (from docs/principles/evaluation.md): Content Accuracy, Pedagogical Approach, Adaptive Teaching, Engagement, Age Appropriateness, Language & Clarity, and Error Handling. Each is scored and weighted for an overall quality score.

Scenario 5 — Additional Features Overview

5.1 Enrichment Profile (Parent-Filled)

📋 DOCUMENTED

Parents can fill out an enrichment profile for their child to help personalize the tutoring experience. The system generates a personality snapshot that the AI tutor uses to adapt its teaching style.

  • Student interests and hobbies
  • Preferred learning style (visual, hands-on, story-based)
  • Academic strengths and areas for improvement
  • Session preferences (length, pace, difficulty)
  • AI-generated personality summary shown to parent

5.2 Voice Input & Text-to-Speech

📋 DOCUMENTED

The tutoring sessions support full voice interaction. Students can speak their answers instead of typing, and the tutor can read responses aloud. Multi-language TTS supports English, Hindi, and Hinglish.

Voice Input

Microphone button ([data-testid='mic-button']) in the chat interface activates speech-to-text. Converts spoken answers to text, eliminating typing for students who prefer speaking.

Text-to-Speech

Powered by Google Cloud TTS API (GOOGLE_CLOUD_TTS_API_KEY configured). Reads tutor responses aloud. Supports English, Hindi (हिंदी), and Hinglish — ideal for Indian K-12 students.

5.3 Pre-Computed Explanation Cards

📋 DOCUMENTED

During book ingestion, the system pre-generates multiple explanation variants for each topic. These are presented as step-by-step explanation cards during Teach Me sessions. If a student wants a different approach, they can request an alternative variant.

Available explanation variants: Everyday analogies · Visual/hands-on explanations · Story-based explanations · Step-by-step procedural breakdowns · Real-world application examples
